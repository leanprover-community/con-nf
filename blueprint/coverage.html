2 Phase 1: The model construction 75%

  • Definition 2.1 (def:phase-1-data)

    Fix a proper type index \(\alpha \). Then phase 1 data at \(\alpha \) consists of:

    1. a group \(\mathrm{All}\) (whose elements we call allowable permutations), with a map \(\varphi : \mathrm{All}\to \mathrm{Str}_\alpha \);

    2. a set \(\tau \) (whose elements we call tangles), equipped with an \(\mathrm{All}\)-action;

    3. a map \(j\) (typed near-litters) from near-litters to \(\tau \), equivariant with respect to the \(\mathrm{All}\)-action on near-litters induced by \(\mathrm{All}\to \mathrm{Str}_{\alpha } \to \mathrm{Str}_{-1}\);

    4. a map \(k\) (typed singletons) from \(\tau _{-1}\) to \(\tau \), again \(\mathrm{All}\)-equivariant w.r.t. the induced action on \(\tau _{-1}\);

    5. for each \(x \in \tau \), a small set \(S_x\) of \(\alpha \)-support-conditions that supports \(x\) under the \(\mathrm{All}\)-action (a designated support for \(x\));

    6. an injection \(\iota : \tau \to \mu \), satisfying the following conditions:

      1. each typed litter \(j(L)\) precedes the typed singletons of all its elements \(a \in L\) — explicitly, \(\iota (j(L)) {\lt} \iota (k(a))\);

      2. each typed near-litter \(k(N)\) which is not a litter comes later than its (typed) litter \(j(N^\circ )\), and after (the typed singletons of) all elements of \(N \mathrel {\triangle }N^\circ \);

      3. for each \(x\) in \(\tau _\alpha \) that is not a typed litter or singleton, \(x\) comes later than all of its designated support – explicitly, for each \((a,A)\) or \((N,A)\) in \(S_x\), we must have \(\iota _\alpha (j(N)), \iota _\alpha (k(a)){\lt}\iota _\alpha (x)\).

      [Note: to see these conditions are not unreasonable, note that each \(x \in \tau \) has \({\lt}\mu \) many things that it must come after, and that the chains of these constraints are of depth at most 4: litters \({\lt}\) atoms \({\lt}\) other near-litters \({\lt}\) everything else. This is said more carefully in lemma 3.26 below.]

  • Definition 2.20 (def:semi-tangle-action)

    Allowable permutations act on semi-tangles.

  • Lemma 2.23 (def:typed-near-litter)

    Any code \((\alpha ,-1,N)\), where \(N\) is a near-litter, gives an element of \(\tau _\alpha \).

  • Lemma 2.24 (def:typed-singleton)

    For any symmetric \(b \in \tau _\beta \), \((\alpha ,\beta ,\{ b\} ) \in \tau _\alpha \). In particular, this works for each atom \(a \in \tau _{-a}\).

  • Lemma 2.25 (lem:small-subsets-closure)

    It should also be evident that \((\alpha ,\beta ,B)\) will always be symmetric if \(|B|{\lt}\kappa \) [take the union of the \(\beta \)-supports of elements of \(B\) and add \(\alpha \) to all the second components of elements of this union]: all small subsets of a type are realized in each higher type.

  • Lemma 2.26 (lem:tangle-action)

    The action of \(\alpha \)-allowable permutations on semi-tangles restricts to an action on tangles. Explicitly, if an \(\alpha \) semi-tangle \(x\) is symmetric, then so is \(\pi x\) for any \(\alpha \)-allowable permutation \(\pi \).

  • Definition 2.28 (def:phase-1-propagates)

    For a proper type index \(\alpha \in \lambda \), given phase 1 data at all levels \(\beta {\lt} \alpha \), then the constructions above provide most components of phase 1 data at level \(\alpha \) — everything except for the embedding \(\iota _\alpha : \tau _\alpha \to \mu \) and its properties.

    [Note that to treat this as a definition, in particular of the “designated supports” component, we are implicitly invoking the axiom of choice — a priori we only know that there exists some support for each new tangle.]

  • Definition 3.1 (def:concrete-phase-1-data)

    For a type index \(\alpha \), concrete phase 1 data at level \(\alpha \) consists of a subset \(\tau \subseteq \mathrm{Pretangle}_\alpha \) and a subgroup \(\mathrm{All}\subseteq G_\alpha \), together with full phase 1 data on these.

    Notes for formalisation:

    • The reason we use subsets from here on is because we will soon impose equalities between these components, and subset equalities are much less nasty to deal with than type equalities.

    • To define this, we almost certainly want to factor out the types \(\tau \) and \(\mathrm{All}\) as parameters of the phase 1 data from the start.

    • We could avoid that “factoring out as a parameter” by assuming this concreteness already in the phase 1 data, i.e. positing \(\tau \) as a subset of pretangles and \(\mathrm{All}\) as a subgroup of \(\mathrm{Str}_\alpha \) from the start. At least for \(\tau \), that seems a bit unnatural — it adds an entirely irrelevant extra assumption throughout phase 1. For \(\mathrm{All}\) it perhaps wouldn’t be so unnatural, since \(\mathrm{All}\) does need to be assumed with at least a homomorphism to \(\mathrm{Str}_\alpha \) anyway.

    The standard concrete phase 1 data at level \(-1\) are the defined \(\tau _{-1} = \mathrm{Pretangle}_{-1}\) and \(\mathrm{All}_{-1} = \mathrm{Str}_{-1}\).

  • Definition 3.10 (def:freedom-of-action-statement)

    Given coherent part 1 data below \(\alpha \), say that freedom of action holds (at \(\alpha \)) if: Given an \(\alpha \)-local bijection \(\pi ^0\) and a specification for each path \(A : \beta \to \alpha \) a bijection of the \(\beta \)-flexible litters, there is a unique allowable permutation \(\pi \in \mathrm{All}_\alpha \) such that:

    1. \(\pi {A}\) extends \(\pi ^0_{A}\) for each \(A\);

    2. for each path \(A : \beta \to \alpha \) and \(A\)-flexible litter \(L\), \(\pi _A(j_\beta (L)) = j_\beta (N)\) for some \(N\) near to \(\chi _A(L)\);

    3. The only exceptions of \(\pi \) are \(\pi ^0\), and everywhere else, \(\pi \) is order-preserving between litters. That is: for each path \(A : -1 \to \alpha \) and litter \(L\), and writing \(N\) for the near-litter \(\pi _A``L\), the restriction of \(\pi _A\) to \(L \setminus \mathrm{dom}(\pi ^0_A)\) is the unique bijection from \(L \setminus \mathrm{dom}(\pi ^0_A)\) to \(N^\circ \setminus \mathrm{rge}(\pi ^0_A)\) which is strictly increasing with respect to the (given/canonical) orders on \(L\) and \(N^\circ \).

    Note: Earlier versions of the note/blueprint stated the 3rd condition in terms of exceptions; recent versions of the note spell it out in more detail and add the “increasing” condition, which is necessary for the uniqueness in the theorem conclusion. If there are any doubts, the current version of Randall’s note should be consulted!

  • Definition 3.13 (def:coding-function)

    For any support \(S\) and tangle \(x\), we can define a function \(\chi _{x,S}\) which sends \(T=\pi (S)\) to \(\pi (x)\) for every \(T\) in the orbit of \(S\) under the action of allowable permutations. We call such functions coding functions .   • Definition 3.5 (def:strong-support)

    A strong support is a support \(S\) with certain additional properties.

    1. If \(((\beta ,-1,x),A,\gamma ) \in S\) then \(x\) is a singleton or a litter.

    2. If \(((\beta ,-1,\{ x\} ),A,\gamma ) \in S\), then \(((\beta ,-1,L),A,\gamma ) {\lt}_S ((\beta ,-1,\{ x\} ),A,\gamma )\), where \(L\) is the litter containing \(x\).

    3. If \(((\beta ,-1,L),A,\delta ) \in S\) and \([L]=f_{\gamma ,\beta }(y)\), where \(\gamma {\lt}\delta \), then there is a \(\gamma \)-support \(T\) of \(y\) such that \(T^{A \setminus \{ \beta \} } \subseteq S\) and each element of the domain of \(T^{A \setminus \{ \beta \} }\) is \(\leq _S ((\beta ,-1,L),A)\). Note that these conditions imply that there is an index-raised version of a strong \(\gamma \)-support of \(y\) included in \(T\).

    Given coherent part 1 data below \(\alpha \), a path \(A : \beta \to \alpha \), and a litter \(L\), say \(L\) is \(A\)-flexible (or that the pair \((L,A)\) is flexible) if \(L\) is not in the range of \(f_{\beta ,\gamma }\), for any \(\gamma {\lt} \alpha \).

    (In particular, if \(\beta = -1\), every such pair is flexible, since \(f_{\gamma ,\beta }\)-maps is only defined for \(\beta \) proper.)
